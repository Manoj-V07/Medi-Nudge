# Medi-Nudge

Medi-Nudge is a full-stack MERN healthcare support application focused on everyday medication adherence, health tracking, AI-assisted summaries, and emergency support workflows.

It includes:
- Secure JWT authentication
- Prescription management with structured timing modes
- Dynamic daily medication reminders with local escalation simulation
- Health notes and periodic summary generation using Groq
- Nearby hospital discovery using OpenStreetMap
- Emergency SOS simulation with caregiver context

## Tech Stack

### Frontend
- React 19 + Vite
- React Router
- Axios
- Tailwind CSS

### Backend
- Node.js + Express
- MongoDB + Mongoose
- JWT authentication
- Groq API (OpenAI-compatible chat completions endpoint)
- OpenStreetMap (Overpass + Nominatim fallback)

## Core Features

## 1) Authentication
- Register and login using email/password
- Passwords are hashed with bcrypt
- JWT token returned on login and stored in localStorage (frontend)
- Protected APIs require `Authorization: Bearer <token>`

## 2) Chat Module (Groq)
- Chat endpoint uses Groq model (`GROQ_MODEL`, default: `llama-3.1-8b-instant`)
- Includes medical safety guardrails
- Restricted advice queries return a fixed safe response

## 3) Prescription Management
- Add/list/delete prescriptions per user
- Prescription timing supports:
  - `standard`: four slots (morning/afternoon/evening/night)
  - `frequency`: N times per day
  - `interval`: every X hours
- Input validation for dosage, duration, and timing structure

## 4) Dynamic Reminder Logic
- No heavy reminder scheduler process required
- Daily reminders are generated dynamically from active prescriptions
- Frontend reminders include:
  - Immediate alert
  - Re-alert after 10 minutes
  - Caregiver escalation simulation after 30 minutes if unacknowledged
- Browser notification + tone + popup used for reminder UX

## 5) Health Dashboard + Summaries
- Add, list, and delete daily health notes
- Summary frequencies:
  - Weekly (7)
  - 15 days
  - Monthly (30)
  - Custom interval
- Summaries generated by Groq and split into:
  - Summary
  - Insights
- Manual summary generation supports force refresh
- Caregiver send is currently simulated via server log + UI message

## 6) Hospital Locator + Emergency SOS
- Uses browser geolocation
- Fetches nearby hospitals from OpenStreetMap using:
  - Overpass API (primary with multiple endpoints)
  - Nominatim (fallback)
- Returns normalized list: name, distance, address, rating (if available)
- Emergency SOS flow:
  - Confirmation prompt
  - Builds emergency payload with location + active medicines
  - Caregiver alert is simulated

## 7) Responsive UI
- Sidebar + page content adapt for mobile/tablet/desktop
- Module screens are arranged for stacked mobile layout
- SOS floating button and dashboard cards are mobile-compatible

## Environment Variables

Create `backend/.env` with:

```env
PORT=5000
MONGO_URI=<your_mongodb_connection_string>
JWT_SECRET=<strong_jwt_secret>
CLIENT_URL=http://localhost:5173

GROQ_API_KEY=<your_groq_api_key>
GROQ_MODEL=llama-3.1-8b-instant
```

Optional for frontend (`frontend/.env`):

```env
VITE_API_URL=http://localhost:5000
```

## API Reference

All routes are prefixed by backend base URL.

## Auth

### POST `/api/auth/register`
Body:

```json
{
  "name": "User",
  "email": "user@example.com",
  "password": "secret123",
  "caregiverPhone": "+919999999999"
}
```

### POST `/api/auth/login`
Body:

```json
{
  "email": "user@example.com",
  "password": "secret123"
}
```

Response includes token and user profile.

## Chat (Protected)

### POST `/api/chat`
Body:

```json
{
  "message": "Tell me about diabetes prevention"
}
```

## Prescription (Protected)

### POST `/api/prescription`
Create prescription.

### GET `/api/prescription`
List user prescriptions.

### DELETE `/api/prescription/:id`
Delete one prescription.

## Reminders (Protected)

### GET `/api/reminders`
Returns dynamically generated reminders for current day.

## Health Notes (Protected)

### POST `/api/health`
Add note.

### GET `/api/health`
List notes.

### DELETE `/api/health/:id`
Delete note.

## Summary (Protected)

### POST `/api/summary/generate`
Body example:

```json
{
  "frequencyType": "weekly",
  "intervalDays": 7,
  "force": true
}
```

### GET `/api/summary`
Get latest stored summary.

## Hospitals (Protected)

### GET `/api/hospitals?lat=<lat>&lng=<lng>&filter=<type>`
`filter` values:
- `general`
- `cardiology`
- `neurology`
- `multi-speciality`

- `userId`, `frequencyType`, `intervalDays`, `lastSummaryDate`, `summaryText`, `insightsText`, `periodStart`, `periodEnd`, `noteCount`

## Security and Safety Notes
- JWT-protected routes for all user-sensitive features
- Password hashing via bcrypt
- Chat prompts restricted from diagnosis/treatment behavior
- Medical and caregiver alerts are simulation-only in current build

## Known Simulation Boundaries
- SOS alert delivery is simulated (console/UI), not integrated with real EMS or SMS gateway
- Caregiver notification for reminders/summaries is simulated
- Hospital ratings/addresses may vary depending on OpenStreetMap completeness

## Troubleshooting

## 401 Unauthorized on protected APIs
- Ensure login succeeded and token is present in localStorage
- Ensure request includes `Authorization: Bearer <token>`

## 502/500 while fetching hospitals
- Backend uses multiple Overpass endpoints and a fallback Nominatim query
- If public map services are rate-limited, retry after a short interval

## Groq summary/chat failures
- Verify `GROQ_API_KEY` and model name
- Check quota/billing limits in Groq account

## MongoDB connection error
- Confirm `MONGO_URI` in `backend/.env`
- Ensure cluster/network access allows the backend host
